# RBAC модель

## Роли
- guest: только публичные страницы и каталог
- student: обучение, отправка заданий, сертификаты
- teacher: управление курсами и заданиями, преподавательская аналитика
- admin: полный доступ, модерация, финансы, фичефлаги, роли

## Ключевые ограничения
1. Регистрация всегда создает роль student.
2. API игнорирует попытки указать admin при регистрации.
3. Изменение ролей доступно только endpoint /api/admin/users/:id/role.
4. Запрет доступа проверяется сервером, а не только UI.

## Матрица доступа (сокращенная)
- /api/auth/register: guest
- /api/student/*: student|teacher|admin
- /api/teacher/*: teacher|admin
- /api/admin/*: admin

## Аудит
Все критические действия фиксируются:
- login/logout
- изменение роли
- бан/разбан
- включение/выключение feature flag
- создание курсов и заданий
